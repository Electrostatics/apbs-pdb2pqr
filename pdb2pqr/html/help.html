This page is designed to give more information and answer questions related to the PDB2PQR server.

PDB and PQR File Format

The PDB2PQR server parses PDB files that are formatted in standard PDB format. At this time, whitespace delimited PDB files are not fully supported and may yield unexpected results. On output, the PQR file is also column-formatted, but with the altLoc, iCode, and chainID fields removed - thus allowing the PQR file to be visualized with either column-parsing standard molecular graphics programs or software (like APBS) that uses whitespace-delimited parsing.

Algorithm Descriptions

The PDB2PQR server allows for two optional algorithms:

  • Ensure that new heavy/hydrogen atoms are not rebuilt too close to existing atoms

    This routine will debump reconstructed atoms, ensuring that they are not being placed within the Van der Waals radii of other nearby atoms. This procedure is carried out by varying the sidechain Χ angle until the steric conflict is resolved.

  • Optimize the hydrogen network

    This routine uses a Monte Carlo algorithm to optimize the global hydrogen bonding network in the protein. As a result of this optimization, the phase of HIS, ASN, and GLN sidechain Χ angles are checked for optimum hydrogen-bonding conformation. Additionally, the routine assigns protonation states to HIS, ASP, and GLU based on optimum hydrogen bonding, local energetics, and model pKa values. Due to the nature of the algorithm, hydrogen optimization is the rate-limiting step of the PDB2PQR server, and should be used cautiously for larger PDB files.
